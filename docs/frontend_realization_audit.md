## 这轮清理后，真正剩下的问题

### 1. 不再是“旧前端假数据污染”

前端层面的旧样例依赖已经基本切断。

### 2. 剩下的主要问题是后端样本不足

例如某些股票在某些时间窗口下：

- 价格序列为 0
- 情绪序列为 0
- 热度序列为 0

这不是前端又回退到旧样例，而是后端本身返回的窗口数据为空或被补零。

### 3. 目前前端仍保留的运行时兜底性质

- 首页：
  - `header/featuredStock` 仍可能是 `mixed`
  - 含义是“真实接口 + 标的元数据”混合，不是旧静态样例
- 单股详情页：
  - `profile/kpis` 仍可能是 `mixed`
  - 含义是“真实接口 + 仓库内股票元数据 + 空态默认值”混合
- 舆情钻取页：
  - 当 `news_items=0` 时保留结构化空态，但不会回退到旧新闻列表
- 回测页 / 联动页 / 总览页：
  - 仍有空态提示逻辑
  - 这是“样本不足保护”，不是“假数据回灌”

### 4. 当前最值得继续修的下一阶段问题

- 后端 `linkage` / `overview` 对部分股票窗口的补零策略还过于生硬，前端虽然已做近端裁剪，但根因仍在数据投影
- 某些分析区块仍以“已生成结果展示”为主，尚未做到更强的交互筛选和多窗口对比
- 如果要继续提升“像成品”的观感，下一步应该优先修后端窗口投影与样本覆盖，再处理前端分包优化与更细的交互动效

## 建议的后续优先级

1. 后端检查哪些标的在当前默认窗口没有真实行情/情绪/热度样本
2. 继续收口 `overview-page` / `linkage-page` / `backtest-page` 的空态表达
3. 若后端样本覆盖提升，再评估是否需要调整默认展示标的
4. 最后再做图表层面的视觉增强和代码分包优化
