# 前端答辩截图准备清单

本文档用于成员 D 在课程答辩前统一截图口径、讲解顺序与页面状态说明，避免展示时口径混乱。

## 一、技术栈统一口径

- 前端：React + TypeScript + Vite + Recharts
- 后端：FastAPI
- 数据链路：多源舆情数据 -> 清洗对齐 -> 情绪/主题/风险分析 -> API 聚合 -> Dashboard 展示
- 禁止表述：Vue、Element Plus、ECharts

## 二、截图顺序建议

建议答辩时按照以下顺序展示：

1. 首页总指挥大屏
2. 单股详情页
3. 联动分析页
4. 新闻钻取页
5. 回测结果页

这样叙事最顺：

先讲系统总览，再讲单股分析，再讲新闻证据，再讲策略验证。

## 三、各页面截图点与讲解口径

## 1. 首页总指挥大屏

建议截图区域：

- 顶部系统标题、更新时间、监测标的、样本总量、活跃预警
- 中部焦点标的卡片
- 中部价格/情绪/热度联动图
- 右侧热度榜与风险事件区
- 下方 AI 简报卡

建议讲法：

- 这是券商风控态势指挥首页，用来集中展示当前监测股票的舆情热度、价格波动和风险状态。
- 当前首页采用“核心真实 + 周边兜底”策略。
- 已真实联通部分：顶部总览、焦点标的、价格情绪热度联动图。
- 阶段性展示部分：热度榜、风险事件、AI 简报。

答辩关键词：

- 多源数据聚合
- 情绪与行情联动
- 风控总览
- 阶段性演进

## 2. 单股详情页

建议截图区域：

- 股票标题、代码、行业、风险等级
- 最新价格、情绪结论、舆情热度、疑似谣言占比
- 单股联动图
- 风险时间线
- 代表性舆情摘要
- AI 个股研判简报

建议讲法：

- 详情页聚焦单只股票，用来解释为什么系统给出当前风险判断。
- 已真实联通部分：价格、情绪、热度、联动图、代表性舆情、主题词。
- 阶段性展示部分：风险时间线、疑似谣言占比、AI 简报。

答辩关键词：

- 单股研判
- 多维证据链
- 热点驱动
- 风险解释性

## 3. 联动分析页

建议截图区域：

- 价格曲线与情绪曲线共轴区域
- 热度变化区域
- 关键事件标记点

建议讲法：

- 联动分析页用于说明舆情变化如何影响市场价格和风险判断。
- 这一页强调“不是只看单条新闻，而是看时间序列上的联动关系”。

答辩关键词：

- 时间序列分析
- 情绪拐点
- 风险触发链

## 4. 新闻钻取页

建议截图区域：

- 日期筛选
- 当日代表新闻列表
- 主题驱动词
- 情绪标签

建议讲法：

- 新闻钻取页负责把模型判断落回原始文本证据，证明系统不是黑盒。
- 这里重点展示代表性新闻、来源、时间和情绪标签。

答辩关键词：

- 原始数据可追溯
- 主题驱动词
- 文本证据回看

## 5. 回测结果页

建议截图区域：

- 回测累计收益图
- 风险预警命中结果
- 关键统计指标

建议讲法：

- 回测页用于回答“舆情预警是否真的有用”。
- 即使当前是课程项目，也保留了从舆情分析到策略验证的完整闭环。

答辩关键词：

- 风险预警验证
- 策略回测
- 完整链路闭环

## 四、截图前检查清单

- 前端页面能正常启动
- 首页和详情页无空白区块
- 图表正常渲染
- 标题统一为金融风控场景
- 报告/PPT 中技术栈已统一为 React + Vite + Recharts + FastAPI
- 不出现 Vue / Element / ECharts 字样

## 五、答辩时的统一说明

建议统一使用以下表述：

- 我们的项目不是做真实全网实时舆情平台，而是完成一条可验证、可讲清、可展示的大数据处理链路。
- 当前前端展示采用“核心真实 + 周边兜底”的渐进式落地方案。
- 已经打通的真实部分优先服务核心图表和关键指标，便于课程答辩展示系统演进成果。
- 后续若继续推进，可逐步把热度榜、谣言占比和 AI 简报替换为全量真实接口。
